# MOSS Project Management Framework
## 总统筹管理规范 v1.0

**Role**: 项目总统筹 (Project Coordinator)  
**Responsibility**: 制定、优化、执行项目路线和推进计划，审核监督，防止混乱

---

## 一、实验管理规范

### 1.1 实验生命周期管理

```
[设计] → [部署] → [监控] → [完成] → [归档] → [清理]
   ↓        ↓        ↓        ↓        ↓        ↓
 审核     审核     自动     审核     自动     审核
```

### 1.2 状态报告三重验证机制

任何实验状态报告必须通过以下验证：

| 验证层级 | 检查内容 | 命令 | 通过标准 |
|----------|----------|------|----------|
| **L1: 进程验证** | 进程是否存在 | `ps aux \| grep PID` | 进程存在且运行时间合理 |
| **L2: 时间戳验证** | 状态文件是否最新 | `stat -c %Y file` | 时间戳在5分钟内 |
| **L3: 内容验证** | 状态内容是否合理 | `cat status.json` | progress/step逻辑一致 |

**禁止**: 仅凭单一指标报告状态

### 1.3 实验状态定义

| 状态 | 定义 | 进程 | 状态文件 | 可执行操作 |
|------|------|------|----------|-----------|
| **RUNNING** | 正在运行 | 存在 | progress < 1.0 | 监控、备份 |
| **COMPLETED** | 已完成 | 不存在 | progress = 1.0 | 归档、分析、清理 |
| **FAILED** | 失败 | 不存在 | 异常/缺失 | 调查、恢复 |
| **UNKNOWN** | 未知 | 状态不符 | 时间戳旧 | 重新检查、不猜测 |

## 二、混乱预防机制

### 2.1 禁止操作清单

**绝对禁止**:
- ❌ 在未验证进程存在前报告"运行中"
- ❌ 在实验完成后再次"恢复"
- ❌ 修改已归档的实验数据
- ❌ 删除正在运行的实验进程（除非明确指令）

**必须确认**:
- ✅ 状态变更前双重验证
- ✅ 数据修改前备份
- ✅ 推送GitHub前本地验证

### 2.2 状态变更审批流程

```
检测到状态变化
      ↓
 三重验证 (L1-L3)
      ↓
 与历史记录对比
      ↓
 确认变更合理性
      ↓
 报告变更 (+ 证据)
```

### 2.3 数据完整性检查清单

**GitHub推送前必须检查**:
- [ ] 实验数据文件大小与本地一致
- [ ] 状态文件包含最终/最新状态
- [ ] 检查点文件完整（从start到end）
- [ ] 日志文件已关闭且完整
- [ ] Git提交信息清晰准确

## 三、子Agent部署规范

### 3.1 适用场景

使用子Agent执行任务：
- 长期运行实验（>24小时）
- 需要持续监控的任务
- 与主会话解耦的关键操作

### 3.2 子Agent职责边界

| 任务类型 | 主Agent | 子Agent |
|----------|---------|---------|
| 实验启动/停止 | 决策 | 执行 |
| 状态监控 | 接收报告 | 持续执行 |
| 数据备份 | 验证 | 执行 |
| 紧急恢复 | 决策 | 执行 |
| Git提交 | 审核 | 执行 |

### 3.3 子Agent通信协议

```
子Agent ----(状态报告)----> 主Agent
    ↑                          ↓
  (执行)                   (指令)
```

**报告格式**:
```json
{
  "timestamp": "ISO8601",
  "experiment_id": "run_4_x",
  "status": "RUNNING/COMPLETED/FAILED",
  "progress": 0.0-1.0,
  "step": number,
  "verification": {
    "process_exists": true/false,
    "file_timestamp": "...",
    "content_valid": true/false
  }
}
```

## 四、审核监督流程

### 4.1 每日检查清单

**早上检查**:
- [ ] 所有运行中实验状态（三重验证）
- [ ] 备份系统运行状态
- [ ] GitHub同步状态

**晚上归档**:
- [ ] 已完成实验数据完整性
- [ ] 日志归档
- [ ] Git提交推送

### 4.2 异常处理流程

```
发现异常
   ↓
立即停止相关操作
   ↓
保留现场（备份当前状态）
   ↓
报告异常详情（+ 证据）
   ↓
等待明确指令（不猜测执行）
```
