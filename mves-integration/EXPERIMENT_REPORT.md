# MVES 实验报告

**实验周期：** 2026-03-26  
**实验版本：** v1 - v4  
**实验目标：** 验证自驱力→持续演化→AGI 路径

---

## 摘要

本实验系列探索了在没有外部任务输入的情况下，系统能否通过内部机制产生持续演化和结构变化。四个版本的迭代揭示了从简单规则演化到驱动系统崩溃的完整图景。

**核心发现：** 演化需要精确的平衡——过于简单的系统无法产生复杂性，过于复杂的系统容易崩溃。

---

## 实验 v1：基础策略演化

### 设计
- 硬编码策略（random/conserve/explore）
- 能量约束 + 死亡机制
- 随机变异

### 结果
| 指标 | 数值 |
|------|------|
| 代数 | 100 |
| 死亡 | 0 |
| 策略跃迁 | 1 次 (random→explore) |
| 最终能量 | -32 |

### 关键发现
✅ 策略可以自发跃迁  
⚠️ 死亡机制未触发（fitness = energy + steps 设计缺陷）

### 结论
基础演化机制成立，但选择压力不足。

### 关键发现
✅ Code as Genome 可行

### 结论
代码演化成立，但需要更智能的变异机制。

## 实验 v4：开放环境 + 驱动系统

### 设计
- 三大驱动（survival/curiosity/control）
- 开放环境（可改造）
- 工具使用系统
- 科学评估框架

### 结果
| 指标 | 数值 | 评估 |
|------|------|------|
| 最终群体 | 2/10 | ❌ |
| 总出生 | 0 | ❌ |
| 总死亡 | 8 | ⚠️ |
| 复杂度 | 1.0 | ❌ |
| 环境改变 | 0 | ❌ |
| 行为熵 | 1.03 | ✅ |

### 关键发现
❌ 驱动失衡（100% curiosity 主导）  
❌ 工具从未使用  
❌ 环境零改造  
✅ 科学评估框架建立

### 失败原因
1. **驱动权重未校准** - curiosity 始终最高
2. **工具无学习机制** - agent 不知何时使用
3. **建造无即时回报** - 缺乏激励
4. **群体太小** - 无多样性

### 结论
复杂系统需要精确校准，"智能"有代价。

## 理论贡献

### 1. 演化平衡原理
> 系统复杂度必须与选择压力匹配

- v1-v3：低复杂度 + 低压力 → 稳定
- v4：高复杂度 + 无压力 → 崩溃

### 2. 驱动校准定理
> 多驱动系统需要动态权重或显式约束

v4 证明：固定权重 + 无学习 → 失衡

### 3. Code as Genome 可行性
> 代码可作为演化基因，但需要：
> - 安全变异机制（AST-based）
> - 回退保护
> - 功能验证

### 4. 科学评估框架
> AGI 实验需要可证伪标准

5 项评估标准：
1. 结构复杂度增长
2. 行为多样性
3. 代际稳定性
4. 环境改变
5. 驱动分化

## 可证伪假设

基于实验结果，提出以下可证伪假设：

### H1：驱动权重影响生存
- **预测：** survival 权重 > 0.7 时，存活率 > 80%
- **检验：** 10 组对照实验（权重 0.3-0.9）

### H2：工具学习需要代际积累
- **预测：** 10 代后工具使用率 > 50%
- **检验：** 追踪工具使用频率

### H3：环境改造需要群体压力
- **预测：** 群体 > 20 时，建造行为出现
- **检验：** 群体大小梯度实验

### H4：代码演化需要语义理解
- **预测：** LLM 变异优于文本替换
- **检验：** 对比 v3 与 LLM 版本
