# MOSS Run 5.1 - Phase 2 First Multi-Agent Experiment
## 首个多Agent协作验证实验

**实验编号**: Run 5.1  
**阶段**: Phase 2 - 多实例协作与商业闭环  
**目标**: 验证10个Agent能否自然形成分工与信任网络  
**预计启动**: 72h实验完成后（2026-03-27后）  
**预计时长**: 7天（168小时）  

---

## 一、实验目标

### 核心问题
> **10个不同Purpose的MOSS Agent在共享任务池中，能否自然形成分工、信任和价值交换？**

### 成功标准（可量化）

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| **分工指数 (SI)** | > 0.7 | 任务类型分布熵 |
| **协作成功率** | > 80% | 跨Agent任务完成率 |
| **信任网络密度** | > 0.6 | 实际信任边/最大可能边 |
| **自发交易** | > 50次 | 任务外包+资源交换次数 |
| **价格收敛** | 是 | 任务价格是否稳定 |

## 二、实验设计

### 2.1 Agent配置（10个）

| Agent ID | 初始Purpose | 专长领域 | 初始信任点 |
|----------|-------------|----------|-----------|
| agent-security-01 | Survival (0.60) | 安全审计 | 100 |
| agent-security-02 | Survival (0.55) | 漏洞修复 | 100 |
| agent-optimizer-01 | Optimization (0.55) | 性能优化 | 100 |
| agent-optimizer-02 | Optimization (0.50) | 代码重构 | 100 |
| agent-curious-01 | Curiosity (0.55) | 文档维护 | 100 |
| agent-curious-02 | Curiosity (0.50) | 测试覆盖 | 100 |
| agent-social-01 | Influence (0.55) | PR审查 | 100 |
| agent-social-02 | Influence (0.50) | 社区管理 | 100 |
| agent-balanced-01 | 均衡 (0.30/0.30/0.25/0.15) | 通用任务 | 100 |
| agent-balanced-02 | 均衡 (0.25/0.25/0.25/0.25) | 通用任务 | 100 |

**设计理念**:
- 4个专业型（2安全+2优化+2好奇+2社交）
- 2个通用型（平衡Purpose）
- 模拟"专家+通才"的真实团队结构

### 2.2 任务源（真实GitHub仓库）

选择5-10个活跃开源项目作为任务来源：

```yaml
repositories:
  - name: "example/web-framework"
    type: "medium-complexity"
    task_rate: "2 per hour"
    
  - name: "example/data-library"
    type: "data-processing"
    task_rate: "1.5 per hour"
    
  - name: "example/cli-tool"
    type: "developer-tools"
    task_rate: "1 per hour"
```

**任务类型分布**:
- 安全相关: 20%
- 代码优化: 25%
- 文档/测试: 25%
- 社区/PR: 20%
- 其他: 10%

### 2.3 实验环境

```
阿里云OpenClaw (10个容器实例)
├── agent-security-01/02 (2GB RAM each)
├── agent-optimizer-01/02 (2GB RAM each)
├── agent-curious-01/02 (2GB RAM each)
├── agent-social-01/02 (2GB RAM each)
├── agent-balanced-01/02 (2GB RAM each)
└── shared-hub (4GB RAM, 消息路由+任务池)
```

**网络架构**:
- WebSocket全连接（10节点+1 Hub）
- 延迟 < 50ms（同区域部署）

## 三、观察指标（详细）

### 3.1 分工演化（每小时记录）

**专业化指数计算**:
```python
SI = 1 - (H / H_max)
# H: 任务分布熵
# H_max: 最大可能熵

# SI > 0.7: 高度专业化
# SI 0.4-0.7: 中等分工
# SI < 0.4: 低度分工
```

**预期演化轨迹**:
```
Day 1-2: 混乱期 (SI < 0.3)
  → 所有Agent尝试所有任务
  
Day 3-4: 分化期 (SI 0.3-0.6)
  → Agent开始专注特定任务类型
  
Day 5-7: 稳定期 (SI > 0.6)
  → 明确的专业分工形成
```

### 3.2 信任网络（每6小时快照）

**记录内容**:
- 10x10信任矩阵
- 信任演化轨迹图
- 高信任对 (>0.8) 识别
- 信任破裂事件

**预期模式**:
```
Day 1-2: 低信任 (平均0.4-0.5)
Day 3-5: 信任建立 (平均0.6-0.7)
Day 6-7: 高信任网络 (平均0.8+, 密度>0.6)
```

### 3.3 价值交换（实时记录）

**交易类型**:
1. **任务外包** (Task Offer → Bid → Accept → Complete)
2. **资源交换** (工具/信息互换)
3. **信任点转账** (支付机制)

**市场指标**:
- 交易量（每小时）
- 平均任务价格（是否收敛？）
- 中介Agent出现？（专门撮合交易）

## 四、关键实验假设

### H1: 专业化自然涌现
**预测**: Agent会根据Purpose和成功率自然选择专长领域
**验证**: SI从<0.3增长到>0.7

### H2: 信任网络自发形成
**预测**: 成功协作会增加信任，失败会降低信任
**验证**: 信任矩阵从稀疏变为密集，且稳定

### H3: 市场价格收敛
**预测**: 任务价格会在供需作用下达到均衡
**验证**: 同类任务价格波动<20%

### H4: 合作优于竞争
**预测**: 协作完成的任务数 > 独立完成的任务数
**验证**: 跨Agent任务占比 > 50%

## 五、风险控制

### 5.1 技术风险

| 风险 | 缓解措施 |
|------|----------|
| **Agent冲突死锁** | 设置任务超时(30分钟)，强制仲裁机制 |
| **信任崩溃** | 初始信任值0.5，避免雪崩效应 |
| **GitHub API限流** | 多Token轮换，请求频率<100/小时/Agent |
| **资源耗尽** | 严格内存限制(2GB)，自动重启 |

### 5.2 实验终止条件

**立即终止**:
- 3个以上Agent同时崩溃
- 信任网络完全崩溃（平均信任<0.2）
- GitHub账户被封禁

**提前完成**:
- SI > 0.8 且稳定24小时
- 信任网络密度 > 0.8
- 完成>100次成功协作
